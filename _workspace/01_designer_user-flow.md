# HI-Sing - 사용자 플로우 설계서

> **노래로 전하는 우리 가족 이야기**
> 마음을 직접 전하기 쑥스럽다면, HI-Sing으로 먼저 전해보세요.

---

## 1. 사용자 여정 (User Journey)

### 전체 여정 맵

```
채널 A (크라우드펀딩):
[인지]          [관심]           [결제]          [이용]            [공유]
  |               |               |               |                |
SNS 광고    →  펀딩 랜딩페이지 → 텀블벅 결제  →  목소리 업로드  →  노래 수신
                서비스 소개       리워드 선택      +요청사항 입력     카톡/링크 공유

채널 B (자체 스토어):
[인지]          [관심]           [결제]          [이용]            [공유]
  |               |               |               |                |
SNS 광고    →  스토어 웹사이트 → 자체 결제    →  목소리 업로드  →  노래 수신
검색 유입      탭별 상세 정보     PG 결제         +요청사항 입력     카톡/링크 공유
```

### 단계별 상세

| 단계 | 사용자 행동 | 감정 상태 | 터치포인트 |
|------|------------|----------|-----------|
| **1. 인지** | SNS 광고에서 서비스 발견 | "이런 게 있어?" 호기심 | 인스타 릴스, 유튜브 숏츠, 페이스북 광고 |
| **2. 관심** | 웹 랜딩페이지에서 샘플 노래 듣기 | "진짜 이렇게 되나?" 기대+의심 | HI-Sing 웹 랜딩페이지 (샘플 오디오, 제작 과정) |
| **3. 확신** | 비포/애프터 비교, FAQ 확인 | "이거 선물하면 감동이겠다" | 웹 랜딩페이지 하단 섹션 |
| **4. 결제** | 텀블벅으로 이동하여 리워드 결제 | "이 가격이면 해볼 만하다" | 텀블벅 결제 페이지 |
| **5. 온보딩** | 텀블벅 결제 후 안내 페이지 확인 | "어렵지 않겠지?" 약간의 긴장 | 텀블벅 결제완료 안내 + 이메일 |
| **6. 제작 요청** | 목소리 업로드 + 요청사항 작성 | "잘 되겠지?" 설렘+불안 | HI-Sing 웹앱 (제작 요청 페이지) |
| **7. 제작 대기** | 이메일로 진행 상황 수신 | "빨리 듣고 싶다" 기대 | 이메일 알림 |
| **8. 수신** | 완성된 노래 듣기 | "와..." 감동 / 소름 | HI-Sing 웹앱 (결과물 페이지) |
| **9. 공유** | 가족에게 선물, SNS 공유 | "자랑하고 싶다" 뿌듯함 | 카카오톡 공유 링크, 인스타 스토리 |
| **10. 재구매** | 다른 가족/이벤트로 재주문 | "다음엔 결혼기념일에" | HI-Sing 웹앱 재방문 |

### 유입 채널별 페이지 구조

HI-Sing은 두 개의 유입 채널을 병행 운영한다.

```
채널 A: SNS 광고 → 크라우드펀딩 랜딩페이지 → 텀블벅/와디즈에서 결제
채널 B: SNS 광고/검색 → 자체 스토어 웹사이트 → 자체 결제 → 바로 제작
```

### 화면별 상세 설계

> 펀딩 시기에 운영. 서비스 소개 → 텀블벅/와디즈 결제로 전환하는 것이 목적.

```
│  먼저 전해보세요              │
│                              │
│  ┌────────────────────────┐  │
│  │ ▶ 샘플 노래 듣기        │  │
│  └────────────────────────┘  │
│                              │
│  ── 이렇게 만들어집니다 ──   │
```

- **목적**: 광고 유입 → 서비스 이해 → 텀블벅 결제 전환

## 4. 사용자 시나리오

### 시나리오 A: 엄마 생신 선물 (오리지널 노래)

> **김수진 (35세, 직장인)**
> 다음 주 엄마 생신인데 특별한 선물을 하고 싶다.

```
Day 0 (발견)
  수진이 인스타그램 피드에서 HI-Sing 광고를 본다.
  "가족 목소리로 만드는 노래" 영상에 호기심이 생긴다.
  → 텀블벅 링크 클릭

Day 0 (결제)
  텀블벅 상세페이지에서 샘플 노래를 듣는다.
  "오리지널 노래 1곡" 리워드(39,000원)를 결제한다.
  → 결제 완료

Day 0 (제작 요청) — 소요 시간: 약 3분
  이메일으로 안내 메시지를 받는다.
  "노래 만들러 가기" 버튼을 누른다.
```

## 5. 핵심 설계 원칙 요약

### "3번 터치" 규칙

```
터치 1: 노래 유형 선택 (오리지널 / 커버)
터치 2: 목소리 파일 업로드
터치 3: 제작 요청 제출

→ 3번의 핵심 행동만으로 주문 완료
  (요청사항 입력은 선택적 보조 행동)
```

### 감성 우선 배치

```
모든 화면에서:
  ✗ "음성 파일을 업로드하세요"
  ✓ "가족의 목소리를 들려주세요"

  ✗ "주문이 접수되었습니다"
  ✓ "당신의 마음이 노래가 되는 여정이 시작됩니다"

  ✗ "음성 파일을 업로드하세요"
  ✓ "마음을 직접 전하기 쑥스럽다면, HI-Sing으로 먼저 전해보세요"

  ✗ "결과물 다운로드"
  ✓ "엄마에게 드리는 노래"
```

### 불안 요소 제거 체크리스트

| 불안 요소 | 해소 방법 |
|----------|----------|
| "어떤 파일을 올려야 하지?" | 섞인 목소리도 OK, 소음 심할 때만 재요청 안내 |
| "노래 퀄리티가 별로면?" | 샘플 미리 듣기 + 1회 무료 수정 |
| "언제 받을 수 있지?" | 예상 완성일 명시 + 중간 알림 |
| "내 목소리 데이터는?" | 동의 기반 학습 데이터 활용 안내 (아래 상세) |
| "환불은 되나?" | 텀블벅 환불 정책 안내 |

### 음성 데이터 활용 정책

| 구분 | 필수 동의 | 선택 동의 |
|------|----------|----------|
| **목적** | 노래 제작 (주문 이행) | AI 모델 학습 (서비스 품질 향상) |
| **대상** | 업로드한 음성 원본 | 비식별화 처리된 음성 특성 데이터 |
| **보관** | 제작 완료 후 30일 뒤 삭제 | 동의 철회 시까지 보관 |
| **미동의 시** | 서비스 이용 불가 | 서비스 이용 가능 (제한 없음) |
| **철회** | - | 언제든 철회 가능 (웹앱 내 설정) |

**학습 데이터 활용 시 원칙:**
- 음성 원본이 아닌 **비식별화된 음성 특성**(음색, 발음 패턴 등)만 학습에 사용
- 학습 데이터로 특정 개인의 목소리를 재현하는 것은 불가능하도록 처리
- 동의 철회 시 해당 데이터는 학습 세트에서 제외
- 이 정책은 텀블벅 캠페인 페이지 및 서비스 이용약관에 명시
